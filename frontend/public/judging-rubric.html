Hebron AI Hackathon

Official Judging Rubric & Scoring Checklist

April 14, 2026 • Palestine Polytechnic University
Instructions for Judges:
  • Score each team from 0 to 15 in every category (4 categories, 60 total).
  • Check each criterion that the team satisfies. Use the checkboxes to ensure every aspect is evaluated.
  • The checklist is a guide — use your professional judgment for the final score per category.
  • Teams use Claude AI as a pair programmer. Judge what they shipped, not whether they wrote every line.
🎨
UI/UX
Visual design, clarity, layout, accessibility and overall user experience
Max 15 pts
Visual Design
Consistent color scheme & typography — unified palette, readable fonts, proper hierarchy
Clean layout & spacing — elements are aligned, whitespace is used effectively, no clutter
Professional polish — icons, imagery, and branding feel intentional and cohesive
User Experience
Intuitive navigation — user can find features without guessing; clear CTAs
Feedback & states — loading indicators, error messages, empty states, success confirmations
Flow & task completion — core user journey is smooth from start to finish
Accessibility & Responsiveness
Mobile-friendly — usable on small screens, touch targets are adequate
Readability & contrast — text is legible, sufficient color contrast for all users
Score / 15
⚡
Frontend
Code quality, responsiveness across devices, performance and interactivity
Max 15 pts
Code Quality
Clean, organized code — readable structure, sensible component breakdown, no spaghetti
Reusable components — proper abstraction, DRY principles applied where reasonable
Performance & Responsiveness
Fast load & interactions — no noticeable lag, efficient rendering, assets optimized
Cross-device compatibility — works on desktop, tablet, and mobile without breaking
Smooth animations/transitions — enhances UX without being distracting or janky
Interactivity & Features
Interactive elements work correctly — forms validate, buttons respond, modals open/close
State management — data flows correctly, UI stays in sync with application state
Error handling in UI — graceful fallbacks, no blank screens or unhandled crashes
Score / 15
🔧
Backend
Architecture, API design, data handling, security and reliability
Max 15 pts
Architecture & API Design
Clear project structure — logical folder layout, separation of concerns, modular design
RESTful / well-designed API — consistent endpoints, proper HTTP methods, meaningful responses
Database design — appropriate schema, relationships, and data modeling for the use case
Security & Reliability
Authentication & authorization — user access is properly controlled and secured
Input validation & sanitization — protected against injection, XSS, and malformed data
Error handling — API returns proper error codes, no stack traces leaked to client
Data & Integration
Data persistence works — CRUD operations function correctly, data survives restarts
External integrations — Claude API or other services integrated cleanly and reliably
Score / 15
💡
Innovation
Originality of the idea, creativity, and how well it addresses a real-world problem
Max 15 pts
Problem & Relevance
Addresses a real problem — solves a genuine need for the city of Hebron or its community
Clear problem statement — team can articulate who benefits and why it matters
Relevance to hackathon theme — aligns with the theme revealed on the day
Creativity & Originality
Novel approach — not a clone of existing solutions; brings a fresh angle
Creative use of AI — leverages Claude in an interesting or unexpected way
Ambition & scope — aims high given the time constraint; shows initiative
Impact & Viability
Potential impact — could make a meaningful difference if developed further
Feasibility — approach is realistic and demonstrates a working proof-of-concept
Score / 15
